# ESP32-C3 QEMU Guide

**English** | [中文](../zh/esp32c3-qemu.md)

## Overview

Espressif maintains a QEMU fork ([espressif/qemu](https://github.com/espressif/qemu))
that supports ESP32-C3 emulation with the following peripherals:

| Peripheral | Status |
|-----------|--------|
| CPU (RISC-V rv32imc) | Supported |
| SPI Flash (2/4/8/16MB) | Supported |
| UART0 | Supported |
| GPIO | Supported |
| Timer Group (TIMG) | Supported (watchdog can be disabled) |
| SHA, AES, RSA, HMAC | Supported |
| Flash Encryption | Supported |
| OpenCores Ethernet MAC | Supported (virtual network) |
| eFuse | Supported |
| WiFi / BLE | **NOT supported** |

**Important**: WiFi and BLE are NOT emulated. Network testing in QEMU uses
the virtual OpenCores Ethernet MAC with `-nic user,model=open_eth`.

## Prerequisites

### Arch Linux

```bash
sudo pacman -S --needed \
    libgcrypt glib2 pixman sdl2 libslirp \
    python cmake ninja gcc git wget flex bison
```

### ESP-IDF + QEMU Installation

```bash
# One-time setup (clones ESP-IDF + installs toolchain + QEMU)
./scripts/setup-esp-env.sh

# Activate environment (every terminal session)
source $HOME/esp/esp-idf/export.sh
```

This installs:
- `riscv32-esp-elf-gcc` — RISC-V toolchain for ESP32-C3
- `qemu-system-riscv32` — Espressif QEMU (ESP32-C3 machine support)
- `esptool.py` — Flash image tool

## Build

```bash
source $HOME/esp/esp-idf/export.sh

# Recommended: unified build
make build-esp32c3-qemu

# Or step-by-step:
cd platform/esp32c3
idf.py -B ../../build/esp32c3-qemu/idf -DRTCLAW_BOARD=qemu set-target esp32c3  # first time only
idf.py -B ../../build/esp32c3-qemu/idf -DRTCLAW_BOARD=qemu reconfigure         # generate compile_commands.json
cd ../..
python3 scripts/gen-esp32c3-cross.py  # generate Meson cross-file
meson setup build/esp32c3-qemu/meson --cross-file build/esp32c3-qemu/cross.ini
meson compile -C build/esp32c3-qemu/meson   # cross-compile core code
cd platform/esp32c3
idf.py -B ../../build/esp32c3-qemu/idf -DRTCLAW_BOARD=qemu build  # link into final firmware
```

## Run on QEMU

### Method 1: Makefile (recommended)

```bash
make run-esp32c3-qemu                  # serial only
make run-esp32c3-qemu GRAPHICS=1      # with LCD display window
```

### Method 2: idf.py wrapper

```bash
cd platform/esp32c3
idf.py -B ../../build/esp32c3-qemu/idf -DRTCLAW_BOARD=qemu qemu monitor
```

Exit with `Ctrl+]`.

This generates a merged flash image and runs:
```
qemu-system-riscv32 -nographic -icount 3 \
    -machine esp32c3 \
    -drive file=build/flash_image.bin,if=mtd,format=raw \
    -global driver=timer.esp32c3.timg,property=wdt_disable,value=true \
    -nic user,model=open_eth
```

### Key QEMU Parameters

| Parameter | Purpose |
|-----------|---------|
| `-machine esp32c3` | ESP32-C3 machine model |
| `-icount 3` | Instruction timing (8ns/insn, ~125MHz) |
| `-drive file=X,if=mtd,format=raw` | Flash image |
| `-nographic` | UART to stdio, no display window |
| `-nic user,model=open_eth` | Virtual Ethernet (user-mode NAT) |
| `-global driver=timer.esp32c3.timg,property=wdt_disable,value=true` | Disable watchdog |
| `-s -S` | GDB server on port 1234, pause at start |

## Debug

### Method 1: idf.py

```bash
# Terminal 1
cd platform/esp32c3
idf.py -B ../../build/esp32c3-qemu/idf -DRTCLAW_BOARD=qemu qemu --gdb monitor

# Terminal 2
cd platform/esp32c3
idf.py -B ../../build/esp32c3-qemu/idf -DRTCLAW_BOARD=qemu gdb
```

### Method 2: Direct GDB

```bash
# Terminal 1
make run-esp32c3-qemu GDB=1

# Terminal 2
cd platform/esp32c3
riscv32-esp-elf-gdb ../../build/esp32c3-qemu/idf/rt-claw.elf -ex 'target remote :1234'
```

## Network in QEMU

WiFi is NOT emulated. Instead, QEMU provides a virtual Ethernet via OpenCores MAC.

Enable with: `-nic user,model=open_eth`

For port forwarding (e.g., expose MQTT port):
```
-nic user,model=open_eth,hostfwd=tcp:127.0.0.1:1883-:1883
```

In the rt-claw ESP32-C3 code, network services that normally use WiFi will
need a compile-time option to use Ethernet when running on QEMU.

## Flash Image Generation

ESP-IDF's `idf.py qemu` handles this automatically. For manual generation:

```bash
cd platform/esp32c3
esptool.py --chip esp32c3 merge_bin \
    --fill-flash-size 4MB \
    -o ../../build/esp32c3-qemu/idf/flash_image.bin \
    @../../build/esp32c3-qemu/idf/flash_args
```

The `flash_args` file is generated by `idf.py build` and contains:
```
0x0     build/bootloader/bootloader.bin
0x8000  build/partition_table/partition-table.bin
0x10000 build/rt-claw.bin
```

## Known Limitations

- WiFi and Bluetooth are not emulated
- Must use `-icount` (no free-running mode)
- RTC watchdog timer not implemented
- Secure Boot not supported in QEMU
- esptool cannot reset chip via RTS over TCP
