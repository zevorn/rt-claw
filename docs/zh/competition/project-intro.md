# RT-Claw：一美元硬件上的嵌入式 AI 助手

> 首届中关村北纬龙虾大赛 · 生产力龙虾赛道

## 你的虾能干什么？

**RT-Claw** 让大语言模型的能力跑在 **¥8（$1）、400KB 内存** 的微控制器上——对芯片说句话，它就能控制硬件。

纯 C99 实现，通过 OSAL 抽象层同时支持 FreeRTOS 和 RT-Thread，已在 ESP32-C3/S3（真机+QEMU）、Zynq-A9、vexpress-A9 四个平台验证通过。AI 可动态调用 30+ 内置工具（GPIO、LCD、音频、网络、定时任务、OTA），无需重新编译固件；多个节点通过蜂群协议自动发现和协作；飞书机器人可远程操控设备；浏览器打开网页即可一键刷机。

## 解决了什么问题？

**AI 困在云端**：当前 AI 助手都跑在云端或手机上，而工厂、农业、智能家居等场景需要低成本边缘 AI。RT-Claw 将 LLM 的 Tool Use 能力部署到 MCU，设备调用云端模型获取决策、本地执行操作，硬件成本从数百元降至几元。

**嵌入式开发慢**：传统流程是写代码→编译→烧录→调试，每次改动都要重来。RT-Claw 用自然语言描述需求，AI 自动编排工具调用，对话即编程。

**设备各自孤立**：单个 IoT 设备无法协作。RT-Claw 内置蜂群智能协议，节点即插即用自动组网，能力共享、任务分发。

## 效果如何？

- 最低 ¥8 硬件运行完整 AI 助手，核心框架 < 400KB RAM、~8000 行 C 代码
- 30+ 工具本地调用 < 10ms，覆盖 GPIO/LCD/网络/音频/调度/OTA
- 4 个平台、2 个 RTOS 验证通过，真机 + QEMU 双通道
- 对 Shell 说"让灯闪 5 次"→ AI 自动调用 `gpio_blink`；说"每小时检查网络"→ 自动创建定时任务
- 天生硬件项目：不是"AI 加硬件"的拼接，而是**从第一行代码就为嵌入式设计的 AI 框架**
